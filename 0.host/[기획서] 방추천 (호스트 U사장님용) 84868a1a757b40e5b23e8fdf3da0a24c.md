# [기획서] 방추천 (호스트/U사장님용)

<aside>
📌

이 문서는 **방추천 기능 - 호스트(U사장님)용** 기획서임. 호스트가 운영 중인 지점 URL을 입력해 맞춤 고객을 확인하고, 메시지 작성 → 알림톡 발송까지 이어지는 흐름을 정의함.

</aside>

## 0) 개요(필수)

<aside>
📌

### **개요(전체 내용 요약)***

- **한 줄 요약**: 호스트가 지점 URL 입력으로 맞춤 고객 리스트를 확인하고 **연락하기(메시지 입력 → 알림톡 발송)**까지 수행하는 콘솔 플로우 제공
- **목적**: 호스트가 조건에 맞는 고객에게 빠르게 컨택하도록 만들어 연락 전환을 만들기 위함
- **배경/가설**: 호스트는 조건이 맞는 고객을 선별해 빠르게 연락하고 싶어하며, 동 기반 리스트/CTA가 제공되면 행동 장벽이 낮아짐
- **핵심 제안**: 호스트 콘솔에서 **운영 지점 URL 입력** → (동) 추출 → 맞춤 고객 리스트 → 연락하기(알림톡 발송) 제공
- **성공 기준(Top 1~2)**: (1) 맞춤 고객 카드 노출 수 (2) 호스트 연락하기 전환율
</aside>

## 1) 배경(선택)

## ✅ 배경(현재 상황/문제/가설)

---

- **현재 상황(As-is)**
    - 호스트가 신규 고객을 찾기 위해 외부 유입/광고에 의존하거나, 조건이 맞는 고객을 선별하기 어려움
- **문제 정의(왜 문제인가)**
    - 맞춤 고객을 빠르게 컨택하기 어려워 연락 전환이 낮아질 수 있음
- **원인 가설(필요한 만큼만)**
    1. ‘맞춤 고객’이 정리된 형태로 제공되면 연락 행동을 더 쉽게 수행함

## 2) 목표/성공 기준(필수)

## ✅ 목표/성공 기준

---

- **핵심 목표**
    - 호스트가 맞춤 고객을 확인하고 **연락하기(메시지 작성/전송)**를 수행하도록 만들기
- **측정 지표(정의가 애매하면 산식까지)**

<aside>
💡

- 호스트 연락 전환율 = (맞춤 고객 카드에서 연락하기 전송 요청 완료) / (맞춤 고객 카드 노출)
</aside>

## 3) 핵심 실행(호스트/U사장님)

## ✅ 핵심 실행(Trigger → Action/CTA → Result)

---

1. **Trigger**: 호스트가 맞춤 고객을 확인하고 컨택하려고 함(호스트 페이지 진입)
2. **Action/CTA**: 휴대폰/ID 입력 → 운영 지점 URL 입력 → 맞춤 고객 리스트 확인 → 연락하기(메시지 입력/전송 요청)
3. **Result**: 조건이 맞는 고객을 빠르게 확인하고 알림톡으로 컨택함

## 4) 범위(In/Out)(합의 필요 시)

## ✅ 범위(In/Out)

---

- **In scope (호스트)**
    - 호스트 조회 플로우(인증 없이 휴대폰/ID/운영지점 URL 입력)
    - URL 크롤링을 통한 (동) 추출 및 크롤링 실패 시 동 직접 입력
    - 맞춤 고객 리스트 노출(정렬: 시간순)
    - 연락하기(메시지 입력 → 전송요청 → DB 저장)
    - 전송요청 완료 고객의 **호스트 단위 disable 처리**
    - DB 기반 **알림톡 발송(내부 처리)**
- **Out of scope**
    - 애드센스 노출/광고 정책(추후)
    - 결제/과금 정책 설계(연락하기 유료화 여부 등)
    - 개인정보/인증/보안 정책 상세(별도 합의 필요)

## 5) 오픈 이슈/결정 필요(호스트)

## ✅ 오픈 이슈(결정 필요)

- (결정) 인증 없이 진행
- (결정) 운영지점 리스트 노출은 파기, **URL 직접 입력**으로 진행
- (결정) URL 크롤링으로 “도보 n분(ㅇㅇ동)” 영역에서 **동 정보** 확보
- (결정) 크롤링 실패 시 **동 직접 입력**으로 대체
- (결정) 유저 매칭: **해당 동이 포함된 구 고객정보 모두 제공**
- (결정) 정렬: 시간순
- (결정) 부족 시: “부족” 안내 + **알림받기 버튼** 제공(추후 유저 생기면 알림)
- (결정) 연락하기: 메시지 입력 → 전송요청 시 DB 저장 → 내부에서 DB 기반 **알림톡 발송**
- (결정) Disable: **해당 호스트에게만** 비활성화
- (추후) 애드센스 연동 시점/정책

## 6) 요구사항(R-*)(호스트)

## ✅ 요구사항(필요할 때만)

---

#### R-1. 접근/인증 정책

- 목적: 빠른 컨택을 위해 진입 장벽 최소화
- 요구사항
    - **인증 없이 진행**(휴대폰 번호는 입력만 받음)

#### R-2. 호스트 기본정보 입력

- 목적: 운영지점/호스트 식별 및 연락하기 발송 처리에 필요한 최소 정보 수집
- 입력 항목
    1. 휴대폰 번호
    2. 운영 중인 지점 URL 입력 (`https://gobang.kr/place/xxxxx`)
    3. 해당 지점 운영중인 U사장님 ID 입력(해당 지점 host로 등록된 ID)
- 버튼/상태
    - 입력값 유효 시 “유저 목록보기” CTA 활성화

#### R-3. 동 정보 확보(크롤링/대체 입력)

- 목적: 지점 기준 동 기반으로 맞춤 고객을 노출하기 위함
- 요구사항
    - “유저 목록보기” 클릭 시 URL 크롤링 로딩 표시
    - `gobang.kr/place` 페이지에서 “도보 n분(ㅇㅇ동)” 영역을 기준으로 **동 정보** 추출
    - 크롤링 실패 시
        - 동 직접 입력 UI 제공
        - 입력 완료 후 리스트 조회 진행

#### R-4. 맞춤 고객 리스트 노출

- 목적: 호스트가 맞춤 고객을 확인하도록 함
- 데이터 소스
    - 구글 스프레드시트: ‘active’ 시트
- 노출 로직
    - **해당 동이 포함된 구 고객정보 모두 제공**
    - 정렬: 시간순
- 리스트/카드 노출 항목
    - 성별, 나이, 희망동
    - 보증금최소(만), 보증금최대(만)
    - 월세최소(만), 월세최대(만)
    - 매물유형
    - 추가요청사항
    - [연락하기] CTA 버튼
- 빈 결과 처리
    - 결과가 부족/없을 경우 “부족” 안내 문구 노출
    - “알림받기” 버튼 제공(추후 유저 생기면 알림 발송/노출)

#### R-5. 연락하기(메시지 입력 → 알림톡 발송)

- 목적: 호스트의 실제 연락 행동(전송 요청)을 만들어 전환 생성
- 요구사항
    - 고객 카드에서 [연락하기] CTA 제공
    - CTA 클릭 시 메시지 입력창 노출
    - “전송요청” → 재확인(컨펌) → 저장
    - 저장된 데이터를 기준으로 내부에서 **알림톡 발송**
    - 전송요청 완료 후 해당 고객 카드는 **해당 호스트에게만 disable** 처리
- 비고
    - 애드센스 연동은 추후(현재 범위 제외)

#### R-6. DB 저장(호스트 사이드)

- 목적: 발송 처리/중복 방지/리포트에 필요한 최소 로그를 남김
- 저장 항목(최소)
    - 호스트 정보: 휴대폰번호, U사장님 ID, 지점 URL
    - 조회 정보: 추출된 동(또는 직접 입력 동), 조회 시간
    - 선택 고객 정보: 고객 식별값 + 카드에 노출된 주요 정보 스냅샷(권장)
    - 메시지 내용
    - 전송요청 시간, 전송 상태(요청/성공/실패 등)

## 7) 유저 플로우(호스트)

## ✅ 유저 플로우(필요한 경우에만)

---

1. 페이지 진입
2. 휴대폰 번호 / 지점 URL / U사장님 ID 입력(인증 없음)
3. “유저 목록보기” 클릭
4. URL 크롤링 → (동) 추출(실패 시 동 직접 입력)
5. 맞춤 고객 리스트 확인(시간순)
6. 고객 카드 “연락하기”
7. 메시지 입력 → 전송요청(컨펌) → 저장
8. 내부에서 알림톡 발송
9. 전송 완료 고객은 해당 호스트에게만 disable
10. 결과 부족 시 “알림받기”로 대기

## 8) 데이터/로그(호스트)

## ✅ 데이터/로그(필요할 때만)

---

- 이벤트 로그(최소)
    - 호스트_조회_시도
    - 호스트_URL입력
    - 호스트_U사장님ID입력
    - 호스트_맞춤고객_조회요청
    - 호스트_크롤링_성공
    - 호스트_크롤링_실패
    - 호스트_동직접입력
    - 호스트_맞춤고객_노출
    - 호스트_연락하기_클릭
    - 호스트_메시지입력
    - 호스트_전송요청
    - 호스트_알림받기_클릭
- 리포트
    - 호스트 전환 퍼널(조회요청→노출→연락하기→전송요청)
    - 동별 조회 대비 전송요청 수

## 9) 검수 체크리스트(호스트)

## ✅ 검수 체크리스트

- 인증 없이도 오남용이 최소화되는 가드레일(입력 검증/레이트 리밋 등)이 정의됨
- URL 크롤링 성공/실패 케이스가 명확히 동작함(실패 시 동 직접 입력 포함)
- “해당 동이 포함된 구 고객정보 모두 제공” 기준이 실제 데이터 형태와 충돌 없이 동작함
- 결과 부족 시 “부족 안내 + 알림받기”가 정상 동작함
- 연락하기(전송요청) 저장 후 알림톡 발송이 정상 동작함
- disable이 “해당 호스트에게만” 적용됨
